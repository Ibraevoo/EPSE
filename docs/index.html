Getting Started

This utility processes standard UniProt FASTA records to transform raw text sequences into organized data blocks.

Basic Execution

To execute the compiled parser binary, pass the source .fasta filepath followed by your designated target output directory:

Execution Syntax:

./parser <path_to_fasta> <output_folder>

Example Command:

./parser uniprot_sprot.fasta /output_data

Expected Standard Output

The console will stream real-time processing confirmations as records are processed:

ID: 1
Accession: Q91G67
Sequence length: 480

Parser Architecture Overview

The core pipeline logic resides within parser/parse.c. It relies on a deterministic single-pass architecture to scan data records efficiently.

How Ingestion Works

  • Line Ingestion: Reads the input stream into a persistent 4096-byte memory buffer via fgets to safe-guard stack space.
  • Header Detection: Isolates rows beginning with a > character to split headers, immediately capturing the structural segments (Accession, Entry Name, Description, and Organism fields).
  • Sequence Accumulation: Captures downstream multi-line string sequences and merges them into a resizing continuous block that doubles its heap capacity whenever a bound limit is reached.

Fixed Size Constraints

To optimize execution speeds, raw fields map to the following explicit buffer boundaries:

  • Accession Strings: Maximum 32 characters.
  • Entry Name Strings: Maximum 32 characters.
  • Organism Identifiers: Maximum 32 characters.
  • Description Field: Maximum 256 characters.

API Reference

Technical specifications for public function entrypoints and struct definitions managed inside the parser library.

Primary Functions

parse_and_save()

Invokes the full storage file parsing loop pipeline.

void parse_and_save(char* filepath, char* folder_name);
  • filepath: Points to the local source FASTA dataset.
  • folder_name: Target filesystem directory for output file generation.

Data Layout Frameworks

Entry

The core structure mapping standard header elements and sequence locations.

typedef struct {
    uint32_t id;
    char accession[32];
    char entry_name[32];
    char description[256];
    char organism[32];
    sequence* seq;
} Entry;

ProteinEmbedding

Allocated space tracking fixed vector dimensions for downstream processing tasks.

typedef struct {
    uint32_t protein_id;
    float embedding[480]; 
} ProteinEmbedding;

Contributing Guide

Contributions are highly appreciated! If you are modifying the codebase or updating the parsing structures, please follow these guidelines:

  1. Fork the repository and develop on a descriptive branch(e.g., feat/refactor-headers).
  2. Ensure any structural adjustments follow the fixed bounds defined in the Entry data model layouts.
  3. Keep modifications focused; individual Pull Requests should solve one explicit issue or feature addition at a time.
